# Handwritten Line Recognition — CNN-BiLSTM-CTC

A complete **handwritten English line recognition** system built entirely from scratch
using PyTorch. No pretrained models, no HuggingFace, no borrowed weights — every
parameter is trained from random initialisation.

---

## Results

| Model | Val CER ↓ | Val WER ↓ | Epochs | Architecture |
|---|---|---|---|---|
| **CRNN V1** | 10.43% | 49.3% | 30 | 4-block CNN + BiLSTM(256) |
| **CRNN V2** | **8.34%** | — | 47 | 5-block CNN + ResBlock + BiLSTM(512) + LayerNorm |

Trained on the **IAM Handwriting Database** (6,482 train · 1,040 val · 2,915 test lines).

> CER = Character Error Rate · WER = Word Error Rate · ↓ = lower is better

## Architecture

```
Input  (B, 1, 32, W)
   │
   ▼
╔══════════════════════════════════════════╗
║  CNN Backbone                            ║
║  Block 1 : Conv(1→64)   + BN + ReLU + Pool(2,2)  ║
║  Block 2 : Conv(64→128) + BN + ReLU + Pool(2,2)  ║
║  Block 3 : Conv(128→256)+ BN + ReLU + Pool(2,1)  ║  ← height-only pool
║  Block 4 : Conv(256→512)+ BN + ReLU              ║
║  Block 5 : ResBlock(512→512)  [V2 only]           ║  ← residual skip
╚══════════════════════════════════════════╝
   │  (B, 512, 4, W/4)  reshape → (W/4, B, 2048)
   ▼
╔══════════════════════════════════════════╗
║  BiLSTM  × 2 layers                     ║
║  hidden = 256 (V1)  /  512 (V2)         ║
║  output = 512 (V1)  / 1024 (V2)         ║
╚══════════════════════════════════════════╝
   │  + LayerNorm(1024) + Dropout(0.2)  [V2 only]
   ▼
Linear → log_softmax → (T, B, vocab+1)
   │
   ▼
CTC Greedy Decode  →  text string
```

> **T** = W / 4 time steps — CNN reduces width by 4× (two MaxPool(2,2) + one MaxPool(2,1))  
> **Blank token** at index 0; real characters start at index 1.

## Commands

### `train.py` — Train the model

```bash
# Train V2 (recommended) — saves to checkpoints/best_model_v2.pth
python train.py --arch v2 --epochs 50 --workers 0

# Train V1 (baseline)
python train.py --arch v1 --epochs 30 --workers 0
```

| Argument | Default | Description |
|---|---|---|
| `--arch` | `v2` | Architecture: `v1` (original) or `v2` (deeper, larger LSTM) |
| `--epochs` | `50` | Maximum training epochs |
| `--batch_size` | `32` | Training batch size (reduce to 16 if VRAM is tight) |
| `--lr` | `1e-3` | Initial learning rate (CosineAnnealingLR with 3-epoch warmup) |
| `--workers` | `4` | DataLoader workers — **use `0` on Windows** |
| `--patience` | `7` | Early stopping patience (epochs without val CER improvement) |
| `--checkpoint_dir` | `checkpoints` | Directory to save `.pth` files |
| `--log` | `logs/training_log.csv` | Per-epoch metrics CSV |
| `--train_csv` | `data/train.csv` | Training split |
| `--val_csv` | `data/val.csv` | Validation split |
| `--img_dir` | `data` | Image root directory |
| `--warmup` | `3` | Linear LR warmup epochs before cosine decay |

### `test.py` — Full test set evaluation

```bash
# Run on test split with V2 model (default)
python test.py

# Use V1 model
python test.py --checkpoint checkpoints/best_model.pth

# Show 20 sample predictions and save per-sample results
python test.py --samples 20 --save logs/test_results.csv
```

**Output includes:**
- Mean / Median / Std CER and WER
- CER distribution histogram (Perfect / Good / OK / Poor / Bad)
- Top-5 best predictions (lowest CER)
- Top-5 worst predictions (highest CER)
- First N sample predictions (ground truth vs. predicted)

| Argument | Default | Description |
|---|---|---|
| `--checkpoint` | `checkpoints/best_model_v2.pth` | Model checkpoint (V1/V2 auto-detected from filename) |
| `--csv` | `data/test.csv` | Test CSV file |
| `--img_dir` | `data` | Image root directory |
| `--batch_size` | `32` | Inference batch size |
| `--workers` | `0` | DataLoader workers |
| `--samples` | `10` | Number of sample predictions to print |
| `--save` | `None` | Optional path to save per-sample CSV |

### `quick_test.py` — Single-image test with camera preprocessing

```bash
# Use default test image (assets/test.png) with V2 model
python quick_test.py

# Test any image
python quick_test.py --image assets/3.png

# Skip preprocessing for clean IAM-style scans
python quick_test.py --image assets/3.png --no_preprocess

# Use V1 model
python quick_test.py --image assets/test.png --checkpoint checkpoints/best_model.pth
```

**Preprocessing pipeline** (auto-activated for RGB/tall images):
```
RGB photo  →  Grayscale  →  CLAHE contrast boost
          →  Otsu binarise  →  Morphological denoise
          →  Auto-crop text region  →  Model input
```

| Argument | Default | Description |
|---|---|---|
| `--image` | `assets/test.png` | Input image path |
| `--checkpoint` | `checkpoints/best_model_v2.pth` | Model checkpoint |
| `--no_preprocess` | `False` | Skip preprocessing (for clean scans) |

## Data Format

CSV files must have exactly two columns (any header names are accepted):

| Column 0 | Column 1 |
|---|---|
| `train/0001.png` | `the quick brown fox` |
| `train/0042.png` | `Hello, World!` |

Images: any format PIL supports · grayscale preferred (RGB auto-converted) · any width
· height is resized to 32 px at runtime preserving aspect ratio.

## Key Design Decisions

| Decision | Detail |
|---|---|
| **CTC blank = 0** | Vocabulary indices start at 1; blank always at 0 |
| **Width-padding collate** | Variable-width images padded to batch max width |
| **Input lengths = W // 4** | Two MaxPool(2,2) + one MaxPool(2,1) → W/4 time steps |
| **No pretrained weights** | All layers init via Kaiming (CNN) + Orthogonal (LSTM) |
| **CosineAnnealingLR** | 3-epoch warmup then cosine decay to lr×0.01 |
| **Real-photo preprocessing** | CLAHE + Otsu + auto-crop bridges IAM↔camera domain gap |
| **V2 auto-detection** | Scripts infer V1/V2 from `_v2` in checkpoint filename |
